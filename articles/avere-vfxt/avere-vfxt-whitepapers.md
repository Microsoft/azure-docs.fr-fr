# <a name="azure-vfxt-for-azure-whitepapers-and-other-resources"></a>Livres blancs Azure vFXT pour Azure et autres ressources

Cet article répertorie des études de cas, des livres blancs technologiques, des infographies et d’autres ressources qui peuvent vous aider à comprendre la façon dont les vFXT pour Azure s’intègrent à votre flux de travail.

## <a name="deploy-agisoft-photoscan-on-azure"></a>Déployer Agisoft PhotoScan sur Azure

**Lien :** [Déployer Agisoft PhotoScan sur Azure avec Avere vFXT pour Azure ou BeeGFS](https://azure.microsoft.com/mediahandler/files/resourcefiles/deploy-agisoft-photoscan-on-azure-with-azere-vfxt-for-azure-or-beegfs/AgiSoft%20PhotoScan%20on%20Azure%20using%20Avere%20vFXT%20or%20BeeGFS.pdf)

Ce guide fournit des instructions détaillées sur l’installation du logiciel de photogrammétrie Agisoft PhotoScan soutenu par un stockage Avere vFXT ou un système de fichiers parallèle BeeGFS. L’auteur, Paulo Marques da Costa, de l’équipe AzureCAT, montre comment installer PhotoScan sur des machines virtuelles Azure. Un stockage haute performance accélère le temps de traitement, et les résultats des tests de point de référence sont inclus. Cet environnement peut être mis à l’échelle en fonction des besoins et prend en charge plusieurs téraoctets de stockage sans sacrifier les performances.

Ce résumé de deux pages contient des informations et diagrammes de base qui expliquent comment utiliser Avere vFXT pour Azure dans différents scénarios.

## <a name="hybrid-cloud-for-file-based-workloads"></a>Cloud hybride pour des charges de travail basées sur des fichiers

**Lien :** [Infographie : Créer un cloud hybride pour des charges de travail basées sur des fichiers](https://azure.microsoft.com/resources/building-a-hybrid-cloud-for-file-based-hpc-workloads/)

Ce graphique montre les composants et les dispositions permettant d’utiliser Avere vFXT pour Azure afin d’ajouter une infrastructure cloud à votre système existant. Utilisez Avere vFXT pour Azure pour augmenter la capacité de calcul avec votre infrastructure locale existante, ou déplacer les données et le calcul vers le cloud de façon temporaire ou permanente.
